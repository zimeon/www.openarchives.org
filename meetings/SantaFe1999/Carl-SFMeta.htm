Santa Fe Metadata Set

The document describes the semantics of the metadata elements that will be used for interoperability among Eprint repositories.  The stated of goal of these elements is simple resource discovery of scholarly Eprints, which are almost exclusively textual documents.  The semantics of the elements is purposely loose with the assumption that values for the metadata elements will be entered by non-professionals (e.g., authors of Eprints).  The transfer syntax of the metadata elements will be XML, the DTD for which is defined in another document.  Finally, except where noted, values for elements are unformatted strings.

Note to the reader: Where the semantics of an element in the Santa Fe Set exactly matches that of the Dublin Core Metadata Element Set, the definition of the Dublin Core element has been used.

Elements that are mandatory are annotated with a [M].  Elements that are optional are annotated with a [O].  Elements that are repeatable are annotated with a [R].  Note that the fact that an elements may be repeated implies that multiple values should not be associated with a single element (e.g.,  associating multiple authors with a single Author tag).

Title [M]

A name given to the document.

Date of Accession  [M]

The date when the document was entered into the repository.  It is assumed that in most cases this date will be created automatically by the repository software rather than entered by a human user.  The date should be encoded in the format YYYY-MM-DD, which is a subset of the guidelines defined in ISO 8601.

Display ID [M] [R]

A URL (Universal Resource Location) identifying a human readable page that provides access to the possible manifestations (e.g., PostScript, TeX) of the document.  For repositories that have only one manifestation per document, this URL may point to that single manifestation.

  Object ID [O] [R]

A unique (within the scope of the repository) identifier of the document.  For some repositories this may be the same as the URL defined by the Display ID, in which case no Object ID should be supplied.

Author [M] [R]

The author or corporate author who is responsible for creating the intellectual content of the document.  Each author may also have an optional institution affiliation. 

Abstract [O]

Text summarizing the contents of the document.

Subject [O] [R]

The topic of the content of the resources expressed as keywords, key phrases or classification codes.

Comment [O] [R]

A free-text value that contains information outside the scope of other defined elements that adds to the discoverability of the document.

Date for Discovery [O] [R]

A date relevant to the document that may aid the user trying to find the document.  A common example of such a date would be an original publication date of a document that was placed in an repository at a later time (i.e., its date of accession is later than its date of publication).  The date should be encoded in the format YYYY-MM-DD, which is a subset of the guidelines defined in ISO 8601.
