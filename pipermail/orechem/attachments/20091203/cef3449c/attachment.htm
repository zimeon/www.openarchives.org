Hi all,<br>
We validated the two Atom feeds that we have now with the <a
 moz-do-not-send="true" href="http://feedvalidator.org/" target="_blank">Feed
Validator</a> and the <a moz-do-not-send="true"
 href="http://african.lanl.gov/ovalnet/validate.jsp" target="_blank">ORE
Atom Resource Map Validator</a>. These validators were mentioned in the
Open Archives' "<a moz-do-not-send="true"
 href="http://www.openarchives.org/ore/1.0/tools">Tools and Additional
Resources</a>" document.<br>
The modified feeds are attached below. The following are the
modifications done:<br>
<br>
1) orechem-article-atom <br>
&nbsp;&nbsp; According to the Feed Validator made small corrections in typos and
the syntax of the &lt;atom:id&gt; and &lt;atom:updated&gt;&nbsp; tags.<br>
&nbsp;&nbsp; According to the ORE Atom Resource Map Validator and looking into
the "<a moz-do-not-send="true"
 href="http://www.openarchives.org/ore/1.0/atom">ORE User
Guide-Resource Map Implementation in Atom</a>" document added two
&lt;atom:category&gt; tags. <br>
<br>
2)Crystal-Eye feed<br>
&nbsp; According to the Feed validator, made corrections in typos and the
syntax of &lt;atom:id&gt; tags. For the sake of validation just used
one of the valid id tag's value present in the feed for all other
&lt;atom:id&gt; tags.<br>
&nbsp;The ORE Atom Resource Map Validator gave a lot of errors. So haven't
made any changes according to that.<br>
<br>
Please take a look into these.<br>
