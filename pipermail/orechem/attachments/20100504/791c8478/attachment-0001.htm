The CIF files should be modelled as instances of File (a subclass of ExperimentObject). Currently, there is no "best practice" for how to link back to the original file (in eCrystals, we use the dc:identifier property.)<br>


<br>
You guessed correctly. The HTML document for the summary/moiety page should link to the instance of ExperimentRun using a &lt;link&gt; tag:<br>
&lt;link rel="alternate" href="..." title="oreChem" type="application/rdf+xml" /&gt;<br>
