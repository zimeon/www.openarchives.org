i've just finished installing a custom-written OAI Dienst-subset
implemention on the local archive i work on (Web Characterization
Repository @ http://purl.org/net/repository)

pros: simple to install, only one file so the code is easier to
understand, no web-server (re)configuration necessary, no pre-requisite
Perl modules, faster than Dienst reference implementation
cons: very archive-specific, lots of code is hard-wired - you need to
know a little more about the OAI protocol before you can use this
