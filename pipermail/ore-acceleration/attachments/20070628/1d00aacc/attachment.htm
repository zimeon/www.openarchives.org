<DIV><FONT face=Arial size=2>Another interesting point is the relation of

much about these guys, but looking at their mission statement there seam to be 
clear overlapping (which shouldn't slow us down):</FONT></DIV>
<DIV>&nbsp;</DIV>
<DIV>"The mission of the Protocol for Web Description Resources (POWDER) Working 
Group is to develop a mechanism through which structured metadata ("Description 
Resources") can be authenticated and applied to groups of Web <A 
href="http://www.w3.org/TR/2004/REC-webarch-20041215/#def-resource">resources</A>. 
This mechanism will allow retrieval of the description resources without 
retrieval of the resources they describe.</DIV>
<DIV>&nbsp;</DIV>

See:</FONT></DIV>
<DIV><FONT face=Arial size=2>Poster about the project: <A

<DIV><FONT face=Arial size=2>Project website <A

<DIV><FONT face=Arial size=2>Data source list <A

<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>

project:</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>DBLP Bibliography Server at Hannover University: <A

<DIV><FONT face=Arial size=2>DBLP Bibliograph Server at Freie University Berlin: 
<A

serving DBLP as Linked Data with dereferencable URIs that support content

data)</FONT></DIV>
<DIV><FONT face=Arial size=2>RDF Book Mashup: <A

dereferenable URIs for each book with a ISBN number and backs these URIs with 
RDF <SPAN lang=EN-US 
style="FONT-SIZE: 10pt; FONT-FAMILY: Arial; mso-fareast-font-family: 'Times New Roman'; mso-ansi-language: EN-US; mso-fareast-language: DE; mso-bidi-language: AR-SA">descriptions</SPAN> 
based on&nbsp;data from Amazon and Google (violating there license agreements, 
but it seams that they don't care as long as it is research).</FONT></DIV>
<DIV><FONT face=Arial size=2>There is a paper about the Book Mashup at <A 
href="http://sites.wiwiss.fu-berlin.de/suhl/bizer/pub/Bizer-ESWC2007-RDFbookmashup.pdf">http://sites.wiwiss.fu-berlin.de/suhl/bizer/pub/Bizer-ESWC2007-RDFbookmashup.pdf</A>&nbsp;and

<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>Another paper that might be relevant for our 
context is the Cool URIs paper <A
