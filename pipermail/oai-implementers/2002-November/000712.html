> tried to figure some out but they have not shown how
> they do it. Do they use a seperate database to store
> the records harvested from the collection database?.
> will that be a good thing?.
> Thank you,
> Anu

If you want to become a Data Provider and you already have
your metadata in a relational database, you do not need an
extra database. All you need is a program (or script) that
maps your existing records to the metadata format you would
like to provide and create XML out of it.
This can be done in real-time, you do not need to store
XML.
