Hi Eric,

I'll be tackling this same issue very soon. My initial thoughts are 1)
sets, as already suggested and 2) do something like:

http://www.domain.com/repo1/oai
http://www.domain.com/repo2/oai

I think with url mapping that could work. And the url could be used to
limit the user's results. (e.g. The set list could be limited to only
return the sets in "his" repository).

>> I can't see anything in the OAI-PMH spec that excludes URL parameters other

>> trouble providing such URLs.
