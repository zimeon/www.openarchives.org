Tim Brody wrote:

> Hi,
> 
> ResumptionToken is *exclusive* - an OAI-compliant harvester must not supply
> a metadataPrefix with a resumptionToken.
> 
> (As has been mentioned in the past by Herbert, if a prefix could be
> supplied, the query could be changed, therefore the result set would be
> inconsistent)
> 
> If a repository requires a metadataPrefix it isn't OAI-compliant, so either
> contact the repository administrator to fix it, or you'll need to code an
> explicit "work-around" for broken implementations.
> 
> All the best,
> Tim.
