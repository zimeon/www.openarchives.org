	For those of you who might be interested, here's a working
tool I put together to further examine the ALA to UNICODE mapping
circulated by Morgan Cundiff.
	
	Attached is an HTML document that includes all of the ALA
characters mapped to UNICODE in NetDev's charconv table.  This
document uses the decimal numeric character references, the
hexadecimal numeric character references, and the HTML character
entity references.

	You need one or more of the UNICODE fonts to view this
document effectively;  see FONTS below.

	This document is best viewed using the Internet Explorer
browser.  Save the attached document on your hard disk.   Open IE,
then click on Files/Open to open the document from wherever you saved
it. Click on Tools/Internet Options/Fonts and then select Arial
UNICODE MS for the Web page font.  Close, and then click Refresh if
necessary to see the UNICODE characters.  (Each character has the
small letter 'o' in front of it in order to test the combining
diacritics; these diacritics simply do not show if there is nothing to
combine with.)

	If you see an anomaly, it could be the font, it could be an
inaccurate mapping in charconv, it could be an error in the browser,
or it could be an error in the Windows UNICODE support.  Depending on
your printer and printer drivers, the characters in this document may
look quite different when printed.  Experiment with one of the other
fonts to see the differences.


FONTS:
	If you don't have it already, you can download the font Arial
UNICODE MS from:
http://office.microsoft.com/downloads/2000/aruniupd.aspx

	You can download the font Bitstream Cyberbit from:
ftp://ftp.netscape.com/pub/communicator/extras/fonts/windows/

 http://home.att.net/~jameskass/
