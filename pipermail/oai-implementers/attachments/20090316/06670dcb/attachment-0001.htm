<p class=MsoNormal>We are working on a project that will take our current
eprints collection from a dedicated eprints repository platform to a new
repository platform where eprints will be one of many collections.&nbsp; I&#8217;m
trying to understand what the OAI implications of this will be and what might
be the impact of harvesters and other service providers.&nbsp; One of the
things is with this broadening of the repository is it will use a different
domain name as currently designed.&nbsp; I note however that this goes against
the OAi Identifier principles as we would want our namespace identifier to
change also.&nbsp; This is currently defined as:<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Courier New"'>&lt;description&gt;<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Courier New"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
&lt;scheme&gt;oai&lt;/scheme&gt;<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Courier New"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
&lt;repositoryIdentifier&gt;eprints.rhul.ac.uk.OAI2&lt;/repositoryIdentifier&gt;<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Courier New"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal>We might in future want it to be:<o:p></o:p></p>

&lt;scheme&gt;oai&lt;/scheme&gt;<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Courier New"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
&lt;repositoryIdentifier&gt;digirep.rhul.ac.uk&lt;/repositoryIdentifier&gt;<o:p></o:p></span></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal><span style='font-size:10.0pt;font-family:"Courier New"'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span style='font-size:12.0pt'>The OAI data provider registration
guidelines (http://www.openarchives.org/data/registerasprovider.html) states a
repository can only have one namespace identifier and a repository may not
change its namespace identifier. &nbsp;Does this mean:<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:12.0pt'><o:p>&nbsp;</o:p></span></p>

style='font-size:12.0pt'><span style='mso-list:Ignore'>a)<span
style='font:7.0pt "Times New Roman"'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><![endif]><span
style='font-size:12.0pt'>It&#8217;s not possible to change the OAI identifier
as specified above?<o:p></o:p></span></p>

<p class=MsoListParagraph style='text-indent:-18.0pt;mso-list:l0 level1 lfo1'><![if !supportLists]><span
style='font-size:12.0pt'><span style='mso-list:Ignore'>b)<span

to remove all registrations for our old repository and register this as a new
repository rather than change the repository information<o:p></o:p></span></p>

<p class=MsoListParagraph style='text-indent:-18.0pt;mso-list:l0 level1 lfo1'><![if !supportLists]><span
