> Guidelines 
 
I fully support all three modifications. As far as I remember the Specs, they 
generally stick to how OAI-PMH can be used for digital libraries and cover the 
respecitve metadata formats. Maybe they could be modified or extended to 
discuss a more generic approach and use of OAI-PMH. 
 
To examplify a possible generic use of OAI-PMH: We use OAI-PMH in the CONESYS 
[1] OSS project, where we define arbitrary metadata repositories in XSD. The 
XML format defined in a XSD is then harvested between the same repositories of 
different CONESYS installations. The XSD-defined metadata format serves as the 
"native" format which can be transformed to other formats like DC to provide a 
standardized metadata interface to other content collectors. In general, 
CONESYS aims to be a middleware and administration software to create content 
networks, by virtually harvesting and distributing any metadata in the context 
of CMS.
